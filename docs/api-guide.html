skill_analyze_url
Analizar URL por cabeceras HTTP
POST

Verifica las cabeceras de seguridad HTTP de cualquier URL según OWASP Top 10 (A05 - Configuración Incorrecta). Detecta ausencia de CSP, HSTS, X-Frame-Options, etc. Devuelve score 0-100 y lista de hallazgos con remediaciones paso a paso.

POST localhost:8001/api/v1/analyze/url
bash
curl -X POST http://localhost:8001/api/v1/analyze/url \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
PowerShell
$body = '{"url": "https://example.com"}'
Invoke-RestMethod -Method POST `
  -Uri "http://localhost:8001/api/v1/analyze/url" `
  -ContentType "application/json" `
  -Body $body | ConvertTo-Json -Depth 5
Python
import requests

r = requests.post(
    "http://localhost:8001/api/v1/analyze/url",
    json={"url": "https://example.com"}
)
data = r.json()
print(data["message"])           # "Score: 40/100. Hallazgos: 4"
print(data["data"]["score"])     # 40
print(data["data"]["findings"])  # lista de hallazgos
CampoTipoReq.Descripción
urlstringrequiredURL completa con http:// o https://
{ "success": true, "data": { "id": 1, "score": 40, "target_type": "url", "findings_count": 4, "severity_summary": { "high": 0, "medium": 4, "low": 0 }, "findings": [{ "rule_id": "OWASP-A05", "title": "Cabecera de seguridad ausente", "severity": "medium", "description": "Falta Content-Security-Policy.", "evidence": "Respuesta HTTP sin Content-Security-Policy", "penalty": 15, "remediation": "Agregar cabecera CSP..." }] }, "message": "Análisis de URL completado. Score: 40/100. Hallazgos: 4" }

skill_analyze_code
Análisis estático de código fuente
POST

Detecta los 10 patrones de vulnerabilidades OWASP Top 10 en código fuente. Incluye: secretos hardcodeados, inyección de código (eval/exec), fallas de autenticación, SSRF, logging insuficiente y más. Soporta Python, JS, Java, PHP, Go, Ruby, C/C++.

POST localhost:8001/api/v1/analyze/code
bash
curl -X POST http://localhost:8001/api/v1/analyze/code \
  -H "Content-Type: application/json" \
  -d '{
    "code": "password = \"12345\"\neval(user_input)\nimport requests",
    "language": "python"
  }'
PowerShell
$body = @{
    code     = "password = '12345'`neval(user_input)"
    language = "python"
} | ConvertTo-Json

Invoke-RestMethod -Method POST `
  -Uri "http://localhost:8001/api/v1/analyze/code" `
  -ContentType "application/json" `
  -Body $body
Python
import requests

code_snippet = """
password = '12345'
api_key = 'sk-secret-abc'
eval(user_input)
"""

r = requests.post(
    "http://localhost:8001/api/v1/analyze/code",
    json={"code": code_snippet, "language": "python"}
)
result = r.json()
print(f"Score: {result['data']['score']}/100")
for f in result["data"]["findings"]:
    print(f"  [{f['severity'].upper()}] {f['rule_id']}: {f['title']}")

skill_analyze_github
Auditar repositorio GitHub completo
POST

Descarga y analiza todos los archivos de código de un repositorio GitHub. Soporta repos públicos y privados (con token). Opcionalmente crea GitHub Issues automáticamente por cada vulnerabilidad encontrada. La evidencia incluye el nombre de archivo exacto.

POST localhost:8001/api/v1/analyze/github
bash
# Repo público
curl -X POST http://localhost:8001/api/v1/analyze/github \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/owner/repo"}'

# Repo privado + crear Issues
curl -X POST http://localhost:8001/api/v1/analyze/github \
  -H "Content-Type: application/json" \
  -d '{
    "repo_url": "https://github.com/owner/private-repo",
    "create_issues": true,
    "github_token": "ghp_xxxxxxxxxxxx"
  }'
Python
r = requests.post(
    "http://localhost:8001/api/v1/analyze/github",
    json={
        "repo_url": "https://github.com/owner/repo",
        "create_issues": False,
        "github_token": None  # o "ghp_xxx" para privados
    }
)
data = r.json()["data"]
print(f"Score: {data['score']}/100")
for f in data["findings"]:
    # evidence incluye el nombre del archivo
    print(f"  {f['evidence'][:60]}")
CampoTipoReq.Descripción
repo_urlstringrequiredURL del repo: https://github.com/owner/repo
create_issuesbooleanoptionalCrear Issues en GitHub (default: false)
github_tokenstring | nulloptionalToken GitHub para repos privados / crear issues
URL del repositorio GitHub

skill_get_report
Obtener reporte por ID
GET

Recupera el reporte completo de un escaneo anterior por su ID. Devuelve todos los hallazgos con evidencia, remediaciones detalladas y resumen de severidades. Ideal para que una IA analice resultados previos.

GET localhost:8001/api/v1/reports/{scan_id}
bash
# Obtener reporte #1
curl http://localhost:8001/api/v1/reports/1

# Listar todos los reportes
curl "http://localhost:8001/api/v1/reports?limit=10"
Python
r = requests.get("http://localhost:8001/api/v1/reports/1")
scan = r.json()["data"]
print(f"Score: {scan['score']}/100")
print(f"Hallazgos: {scan['findings_count']}")
print(f"Severidades: {scan['severity_summary']}")

skill_compare_scans
Comparar dos escaneos
POST

Compara dos escaneos para evaluar la evolución de la seguridad. Devuelve: delta de score, vulnerabilidades resueltas (fixed_rules), nuevas regresiones (added_rules) y un resumen en texto natural para que la IA lo interprete directamente.

POST localhost:8001/api/v1/reports/compare
bash
curl -X POST http://localhost:8001/api/v1/reports/compare \
  -H "Content-Type: application/json" \
  -d '{"scan_id_left": 1, "scan_id_right": 2}'
{ "success": true, "data": { "score_delta": +25, "findings_delta": -3, "fixed_rules": ["OWASP-A02", "OWASP-A03"], "added_rules": [], "persistent_rules": ["OWASP-A01"], "summary": "⬆️ Mejoró 25 puntos | ✅ Resueltas: OWASP-A02, OWASP-A03" }, "message": "⬆️ Mejoró 25 puntos | ✅ Resueltas: OWASP-A02, OWASP-A03" }
ID escaneo base (izquierda)
ID escaneo a comparar (derecha)

skill_dashboard_stats
Estadísticas globales del sistema
GET

Devuelve estadísticas globales: total de escaneos, findings, score promedio, distribución de severidades (high/medium/low) y tendencia de los últimos 10 scans. El campo message incluye el nivel de riesgo general (🟢 BAJO / 🟡 MEDIO / 🔴 ALTO).

GET localhost:8001/api/v1/stats
bash
curl http://localhost:8001/api/v1/stats
🔐
Autenticación (opcional)
Por defecto REQUIRE_AUTH=false — sin token para facilitar desarrollo
💡 Tokens pre-configurados: demo-token-12345 (usuario: demo) y admin-token-67890 (usuario: admin). Si REQUIRE_AUTH=false, puedes hacer peticiones sin token.
bash — con auth
# 1. Generar token nuevo (en el sistema web, puerto 8000)
curl -X POST http://localhost:8000/api/token -F "username=tuusuario"
# → {"token": "uuid-generado", "user": "tuusuario"}

# 2. Usar token en la API (header X-API-Key)
curl http://localhost:8001/api/v1/stats \
  -H "X-API-Key: demo-token-12345"

# Activar auth en .env
REQUIRE_AUTH=true

OpenAI / GPT-4o
Function Calling
Python
import openai, requests, json

# Cargar skills automáticamente
skills = requests.get(
    "http://localhost:8001/api/v1/skills"
).json()["data"]["skills"]

tools = [{
    "type": "function",
    "function": {
        "name": s["name"],
        "description": s["description"],
        "parameters": s["input_schema"]
    }
} for s in skills]

client = openai.OpenAI()
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role":"user",
               "content":"Analiza https://example.com"}],
    tools=tools, tool_choice="auto"
)

LangChain
StructuredTool
Python
from langchain.tools import StructuredTool
import requests

BASE = "http://localhost:8001"

def analyze_code(code: str, language: str=None):
    """Analiza código fuente OWASP Top 10."""
    r = requests.post(
        f"{BASE}/api/v1/analyze/code",
        json={"code": code, "language": language}
    )
    return r.json()["message"]

def get_stats():
    """Estadísticas globales de seguridad."""
    r = requests.get(f"{BASE}/api/v1/stats")
    return r.json()["message"]

tools = [
    StructuredTool.from_function(analyze_code),
    StructuredTool.from_function(get_stats),
]
